IoT & Cyberphysische Systeme

Was ist IoT?
Definition
Internet of Things: physische Objekte werden mit einer digitalen/virtuellen Repräsentation verbunden und können Daten senden, empfangen oder verarbeiten.
Sensoren
Erfassen Daten aus der realen Welt, z. B. Temperatur, Bewegung, Licht, Druck oder CO₂.
Aktoren
Beeinflussen die reale Welt, z. B. Motor, Relais, Lampe, Ventil oder Lautsprecher.
Vernetzung
Kommunikation über WLAN, Bluetooth, Ethernet, Mobilfunk, MQTT usw.
Cloud
Speichert, analysiert und visualisiert Daten, z. B. Dashboards oder KI-Auswertung.
Was ist CPS?

Cyberphysische Systeme verbinden Informatik, Software, Sensoren, Aktoren und mechanische Komponenten.

Kernidee
Digitale Rechenprozesse wirken direkt auf physikalische Prozesse zurück.
Komponenten
Sensoren + Aktoren + eingebettete Systeme + Software + HMI/Bedienoberfläche.
Hierarchie
CPS → IIoT → Industrie 4.0
Automatisierungsgrade
GradBeschreibungBeispiel
1. ÜberwachungKontinuierliche Datenerfassung ohne Eingriff.Temperatur einer Maschine anzeigen.
2. OptimierungAutomatische Anpassung von Parametern.Heizung passt sich an Außentemperatur an.
3. AutonomieSystem trifft selbstständig Entscheidungen.Autonomes Fahrzeug wechselt Spur.
4. SelbstheilungSystem erkennt und behebt Fehler automatisch.Server startet fehlerhaften Prozess neu.
Prüfungstipp
IIoT vs. Industrie 4.0: IIoT ist IoT in der Industrie. Industrie 4.0 baut darauf auf und ergänzt KI, Cloud Computing, Big Data und intelligente Fabrikprozesse.

Sensorik

Definition Sensor
Ein Sensor erfasst physikalische, chemische oder biologische Größen. Das Eingangssignal wird durch physikalische/chemische Effekte in eine Ausgangsgröße umgewandelt, meistens in ein elektrisches Signal.
Aktive Sensoren

Erzeugen selbst elektrische Signale, z. B. Spannung oder Ladung. Sie brauchen keine externe Hilfsenergie zur Signalerzeugung.

Thermoelement
Zwei Metalle erzeugen bei Temperaturdifferenz eine Spannung.
Piezoelement
Kraft/Druck auf Kristall oder Keramik erzeugt ein Ladungssignal.
Vorteile: günstig, robust, keine separate Signalenergie nötig.
Passive Sensoren

Verändern ihre elektrischen Eigenschaften durch Umgebungsänderungen und benötigen Hilfsenergie.

Resistiv
Widerstand ändert sich, z. B. NTC/PTC oder Dehnungsmessstreifen.
Kapazitiv
Kapazität ändert sich, z. B. Feuchtigkeit oder Berührung.
Induktiv
Elektromagnetisches Feld erkennt leitfähige Objekte berührungslos.
Vorteile: oft präziser, größere Reichweite, gut standardisierbar.

Ein CO₂-Sensor ist ein chemischer Sensor. Er benötigt meist eine Versorgungsspannung und liefert z. B. ein normiertes Ausgangssignal wie 0–10 V oder 4–20 mA.

Signalverarbeitung & Messkette

Ablauf einer Messkette
  1. 1
    Sensor/Aufnehmer erfasst die physikalische Größe und erzeugt ein Signal.
  2. 2
    Signalverstärker verstärkt schwache Signale und verbessert Lesbarkeit.
  3. 3
    Messumformer wandelt das Signal in ein normiertes Einheitssignal.
  4. 4
    ADC wandelt analoges Signal in digitales Signal für Mikrocontroller/Computer.
  5. 5
    Weiterverarbeitung durch Mikrocontroller, Computer, Anzeige oder Cloud.
Messumformer

Ein Messumformer macht aus unterschiedlichen Sensorsignalen standardisierte Signale.

Stromsignale: 0…20 mA oder 4…20 mA Spannungssignal: 0…10 V Ratiometrisch: 0,5…4,5 V bei 5 V Versorgung
Datenblatt lesen
Messbereich
Kleinster bis größter messbarer Wert.
Auflösung
Kleinste erkennbare Änderung, z. B. 0,056 mm.
Ausgangssignal
Z. B. 0–10 V oder 4–20 mA. Hat der Sensor Versorgungsspannung? Dann meist passiv.

Regelung, Steuerung & Aktoren

Steuerung

Eine Steuerung läuft ohne Rückkopplung. Das System weiß nicht, ob das gewünschte Ergebnis wirklich erreicht wurde.

Beispiel
Lichtschalter: Schalter an → Strom fließt. Ob die Lampe wirklich leuchtet, wird nicht geprüft.
Regelung

Eine Regelung arbeitet mit Rückkopplung. Ist-Wert wird gemessen, mit Sollwert verglichen und bei Abweichung korrigiert.

Beispiel
Heizung: gewünschte Temperatur = Sollwert. Thermometer misst Ist-Wert. Heizung regelt nach.
Regelkreis – typische Reaktionen
ReaktionWann?Was passiert?
AusgleichKleine AbweichungSystem korrigiert und stellt Sollwert wieder her.
AlarmZu starke Abweichung oder kritischer TrendWarnsignal wird ausgelöst.
AbschaltungGefahr oder Alarm zu lange ignoriertSystem schaltet ab, um Schäden zu vermeiden.
Sensor vs. Aktor: Sensor misst und liefert Daten. Aktor empfängt ein Signal und führt eine Aktion aus, z. B. Relais, Motor, Ventil, Lampe oder Lautsprecher.

Pullup-Widerstand

Pin liegt standardmäßig auf HIGH. Beim Tastendruck wird er mit GND verbunden → LOW.

5V ──[10kΩ]── Pin ── Taster ── GND Taster offen = HIGH Taster gedrückt = LOW
Pulldown-Widerstand

Pin liegt standardmäßig auf LOW. Beim Tastendruck wird er mit 5V verbunden → HIGH.

5V ── Taster ── Pin ──[10kΩ]── GND Taster offen = LOW Taster gedrückt = HIGH
Warum Pull-Widerstände?
Ohne Pullup/Pulldown schwebt der Pin. Er ist dann nicht eindeutig HIGH oder LOW. Typisch sind 10 kΩ.

PWM – Theorie

Was ist PWM?
Pulsweitenmodulation: Ein digitales Signal wird sehr schnell ein- und ausgeschaltet. Über den Tastgrad entsteht ein bestimmter Mittelwert bzw. eine wahrgenommene Helligkeit oder Geschwindigkeit.
Grundbegriffe
Periode
Ein kompletter Ein-Aus-Zyklus.
Frequenz
Wie oft der Zyklus pro Sekunde passiert. Einheit: Hz.
Tastgrad
Verhältnis von Einschaltzeit zu Periodendauer in Prozent.
Arduino
Meist ca. 490 Hz, Pins 5 und 6 ca. 980 Hz.
Formeln
Tastgrad = Pulsdauer / Periodendauer U_m = U_max × Tastgrad U_eff = U_max × √Tastgrad
Beispiele mit Umax = 5 V
TastgradU_mU_eff
25%5 V × 0,25 = 1,25 V5 V × √0,25 = 2,5 V
50%5 V × 0,5 = 2,5 V5 V × √0,5 ≈ 3,5 V
75%5 V × 0,75 = 3,75 V5 V × √0,75 ≈ 4,3 V
analogWrite(pin, 0–255): 0 = 0%, 128 ≈ 50%, 255 = 100%.

MQTT

Was ist MQTT?
Message Queuing Telemetry Transport ist ein leichtgewichtiges Protokoll für IoT. Es nutzt das Publish/Subscribe-Modell mit einem zentralen Broker.
Publish/Subscribe
MerkmalPub/SubClient-Server
KopplungLocker entkoppeltDirekte Verbindung
VermittlerBrokerKein Zwischenknoten
Skalierbarkeithochbegrenzter
MQTT-Broker

Der Broker nimmt Nachrichten von Publishern an, filtert sie anhand von Topics und verteilt sie an Subscriber.

Beispiel: Mosquitto als MQTT-Broker in Smart-Home- oder Industrie-Systemen.
Topics & Wildcards

Topics sind hierarchisch aufgebaut und werden mit / getrennt.

smart_home/wohnzimmer/temperatur smart_home/schlafzimmer/licht/status smart_home/sensoren/{raum}/co2
WildcardBedeutungBeispiel
+Genau eine Ebenesmart_home/+/temperatur
#Alle folgenden Ebenensmart_home/#
Quality of Service
LevelNameZustellungEinsatz
QoS 0At most onceHöchstens einmal, keine Bestätigung.Unkritische Sensordaten.
QoS 1At least onceMindestens einmal, kann doppelt ankommen.Alarme, wichtige Ereignisse.
QoS 2Exactly onceGenau einmal, sicherste Stufe, mehr Overhead.Kritische Steuerbefehle.
Sicherheit
TLS
Verschlüsselte Übertragung, oft Port 8883.
Auth
Benutzer/Passwort oder Zertifikate.
Zertifikate
Sichere Identitäten für Geräte.
Funktionen
Retain
Broker speichert letzte Nachricht für neue Subscriber.
LWT
Last Will: Nachricht bei unerwartetem Verbindungsverlust.
Keep Alive
Heartbeat zur Überwachung der Verbindung.
MQTT vs. HTTP
MQTT ist für IoT oft besser geeignet: kleinerer Overhead, persistente Verbindung, bidirektionale Kommunikation und gut für ressourcenarme Geräte.

Arduino-C++ Programmstruktur

Jedes Arduino-Programm heißt Sketch und besitzt zwei Pflichtfunktionen: setup() und loop().

Grundgerüst jedes Arduino-ProgrammsC++ / Arduino
// Globale Konstanten und Variablen
#define LED_PIN 13          // kein Semikolon bei #define
const int buttonPin = 2;
int counter = 0;

void setup() {
  pinMode(LED_PIN, OUTPUT);
  pinMode(buttonPin, INPUT);
  Serial.begin(9600);
}

void loop() {
  // Hauptprogramm läuft endlos
}
setup()
Läuft genau einmal beim Start oder Reset. Hier werden Pins und Serial Monitor vorbereitet.
loop()
Läuft endlos, solange der Arduino Strom hat. Hier steht das eigentliche Programm.
Häufiger Fehler
Setup() ist falsch. Arduino/C++ unterscheidet Groß- und Kleinschreibung: richtig ist setup().

Eigene Funktionen

Funktionen vermeiden Wiederholung und machen den Code übersichtlicher.

void ledBlinken(int pin, int mal) {
  for (int i = 0; i < mal; i++) {

    delay(300);
    digitalWrite(pin, LOW);
    delay(300);
  }
}

float cmZuMeter(float cm) {
  return cm / 100.0;
}

int summe(int a, int b) {
  return a + b;

void loop() {

Serial Monitor

Der Serial Monitor ist wichtig zum Debuggen. Du kannst Texte und Messwerte am Computer anzeigen.

Serial MonitorC++ / Arduino
void setup() {
  Serial.begin(9600);
}

void loop() {
  int sensor = analogRead(A0);

  Serial.print("Wert: ");

print vs. println

  }
}

Taster mit INPUT_PULLUP

Interner Pullup – Logik invertiertC++ / Arduino
#define LED    13
#define TASTER 2

void setup() {
  pinMode(LED, OUTPUT);
  pinMode(TASTER, INPUT_PULLUP);
}

void loop() {
  int zustand = digitalRead(TASTER);

  // gedrückt = LOW, nicht gedrückt = HIGH
  if (zustand == LOW) {
    digitalWrite(LED, HIGH);
  } else {
    digitalWrite(LED, LOW);
  }
}

Toggle: Taster schaltet LED um

LED ein/aus mit TasterC++ / Arduino
#define LED    13
#define TASTER 2

bool ledZustand = false;
int letzterZustand = LOW;

void setup() {
  pinMode(LED, OUTPUT);
  pinMode(TASTER, INPUT);
}

void loop() {
  int aktuell = digitalRead(TASTER);

  if (aktuell == HIGH && letzterZustand == LOW) {
    ledZustand = !ledZustand;
    digitalWrite(LED, ledZustand);
    delay(50);
  }

  letzterZustand = aktuell;
}

Poti steuert LED-Helligkeit

analogRead zu analogWriteC++ / Arduino
#define POTI A0
#define LED  9

void setup() {
  pinMode(LED, OUTPUT);
  Serial.begin(9600);
}

void loop() {
  int potiWert = analogRead(POTI);     // 0–1023
  int helligkeit = potiWert / 4;        // 0–255

  analogWrite(LED, helligkeit);

  Serial.print("Poti: ");
  Serial.print(potiWert);
  Serial.print("  Helligkeit: ");
  Serial.println(helligkeit);

  delay(100);
}
Merken
analogRead() liefert 0–1023. analogWrite() braucht 0–255. Darum: helligkeit = potiWert / 4;

Ultraschallsensor HC-SR04

Aufbau & Messprinzip
PinFunktionArduino
VCC5-V-Versorgung5V
GNDMasseGND
TRIGLöst Ultraschallpuls ausDigitaler Ausgang, z. B. 9
ECHOGibt Pulsdauer zurückDigitaler Eingang, z. B. 10
Formel
TRIG 10 µs auf HIGH → Sensor sendet Ultraschall. ECHO bleibt HIGH, bis Echo zurückkommt.
Distanz = (Zeit × Schallgeschwindigkeit) / 2. Praktisch: distanz = dauer / 58.2 in cm.

Vollständiges Ultraschallprogramm

HC-SR04 Distanz messenC++ / Arduino
#define TRIG_PIN 9
#define ECHO_PIN 10

void setup() {
  pinMode(TRIG_PIN, OUTPUT);
  pinMode(ECHO_PIN, INPUT);
  Serial.begin(9600);
}

void loop() {
  digitalWrite(TRIG_PIN, LOW);
  delayMicroseconds(2);

  digitalWrite(TRIG_PIN, HIGH);
  delayMicroseconds(10);
  digitalWrite(TRIG_PIN, LOW);

  long dauer = pulseIn(ECHO_PIN, HIGH);
  float distanz = dauer / 58.2;

  Serial.print("Distanz: ");
  Serial.print(distanz);
  Serial.println(" cm");

  delay(500);
}

Abstandswarner: LED-Alarm

Ultraschall + LED-AlarmC++ / Arduino
#define TRIG_PIN  9
#define ECHO_PIN  10
#define LED_ROT   13
#define GRENZWERT 20

float messeDistanz() {
  digitalWrite(TRIG_PIN, LOW);
  delayMicroseconds(2);
  digitalWrite(TRIG_PIN, HIGH);
  delayMicroseconds(10);
  digitalWrite(TRIG_PIN, LOW);

  long dauer = pulseIn(ECHO_PIN, HIGH);
  return dauer / 58.2;
}

void setup() {
  pinMode(TRIG_PIN, OUTPUT);
  pinMode(ECHO_PIN, INPUT);
  pinMode(LED_ROT, OUTPUT);
  Serial.begin(9600);
}

void loop() {
  float dist = messeDistanz();

  Serial.print("Distanz: ");
  Serial.print(dist);
  Serial.println(" cm");

  if (dist < GRENZWERT) {
    digitalWrite(LED_ROT, HIGH);
    Serial.println("ZU NAH!");
  } else {
    digitalWrite(LED_ROT, LOW);
  }

  delay(300);
}
Wichtig
delayMicroseconds(10) sind Mikrosekunden, nicht Millisekunden. pulseIn() gibt Mikrosekunden zurück. Für dauer besser long statt int nutzen.

Codefehler erkennen & korrigieren

In der Prüfung bekommst du oft fehlerhaften Code. Diese Fehler kommen sehr häufig vor.

1. Semikolon fehlt oder ist falsch gesetzt

Falsch
int x = 5
digitalWrite(13, HIGH)
if (x == 5); {
  Serial.println("ok");
}
Richtig
int x = 5;
digitalWrite(13, HIGH);
if (x == 5) {
  Serial.println("ok");
}

2. = statt ==

Falsch
if (zustand = HIGH) {
  // Zuweisung, kein Vergleich
}
Richtig
if (zustand == HIGH) {
  // Vergleich
}

3. pinMode oder Serial.begin vergessen

Falsch
void setup() {
  // pinMode und Serial.begin fehlen
}

void loop() {
  digitalWrite(13, HIGH);
  Serial.println("Test");
}
Richtig
void setup() {
  pinMode(13, OUTPUT);
  Serial.begin(9600);
}

void loop() {
  digitalWrite(13, HIGH);
  Serial.println("Test");
}

4. Falscher Datentyp beim Ultraschall

Falsch
int dauer = pulseIn(ECHO, HIGH);
int distanz = dauer / 58.2;
Richtig
long dauer = pulseIn(ECHO, HIGH);
float distanz = dauer / 58.2;

5. Falscher PWM-Pin oder falscher Wert

Falsch
analogWrite(2, 128);   // Pin 2 kein PWM
analogWrite(9, 300);   // Wert > 255
Richtig
analogWrite(9, 128);   // PWM-Pin
analogWrite(9, 255);   // Maximalwert

6. for-Schleife als Endlosschleife

Falsch
for (int i = 0; i < 5; i--) {
  // i wird kleiner statt größer
}
Richtig
for (int i = 0; i < 5; i++) {
  // i zählt hoch
}
